# Blindtrain

Веб-тренажёр слепой печати на ЙЦУКЕН с элементами геймификации и контролем позы рук через веб-камеру.

## Про что это

Пользователь проходит 8 уроков от домашнего ряда `фыва олдж` до свободной практики. Печатает прямо по эталонному тексту — отдельного поля ввода нет. Каждая ошибка сразу видна (красный символ) и снижает точность в реальном времени. Виртуальная клавиатура окрашена в цвета пальцев, SVG-схема рук показывает, каким пальцем жать следующий символ.

За завершённую сессию начисляется XP (`wpm × accuracy / 100`). Уровни открывают кастомизацию: темы, цвета, шрифты, титулы, возможность загрузить свой фон.

Камера через MediaPipe Hands проверяет, что обе руки на клавиатуре, и подсказывает, если поза неправильная (руки перепутаны, слишком близко/далеко, сильный наклон). Без двух рук в кадре печать блокируется — экран мигает красным.

## Стек

Flask + SQLAlchemy + Flask-Login на бэке, vanilla JS на фронте (без фреймворков), MediaPipe Hands для CV, Pillow для валидации загружаемых картинок.

## Уроки

| Урок | Что осваиваем |
|---|---|
| 1 | Левая рука: `фыва` |
| 2 | Правая рука: `олдж` |
| 3 | Обе руки: `фыва олдж` |
| 4 | + Верхний ряд левой: `йцуке` |
| 5 | + Верхний ряд правой: `нгшщ` |
| 6 | + Нижний ряд: `ячсмит ьбю` |
| 7 | + Цифры и пунктуация |
| 8 | Свободная практика |

Правила доступа: 1-3 строго по порядку, 4-7 — в свободном порядке после 1-3. Урок 8 — после 1-3 + минимум 3 из 4-7 + средняя точность за последние 5 сессий ≥ 85%. Упражнение засчитывается при точности ≥ 90%; иначе повторяешь заново.

## Геймификация

**XP и уровни.** `XP = wpm × accuracy / 100`. Порог уровня `N` — `N × 100` XP. Потолка нет.

**Streak.** Любая завершённая сессия засчитывает день активности (UTC). Пропуск полного дня — серия сбрасывается, рекорд сохраняется отдельно.

**18 достижений** в 4 категориях: прогресс, серии (3/7/14/30 дней), скорость (100/150/200/300 зн/мин с точностью ≥ 90), точность и упорство (без ошибок, три идеальных подряд, 10/50/100 сессий и т.д.).

**Рейтинг** (`/leaderboard`). Две вкладки — по уровню и по лучшей скорости (с фильтром `accuracy ≥ 90` — анти-читер). Топ-20 + позиция текущего пользователя, если не в топе. Есть приватный режим («Аноним» в рейтинге — место сохраняется).

## Кастомизация с unlock'ами

Сервер отдаёт только те опции, которые пользователь уже разблокировал; остальные показываются серыми с замком и уровнем разблокировки.

| Категория | Уровни |
|---|---|
| Темы | Светлая (1), Тёмная (7) |
| Цвет текста | default (1), синий (3), зелёный (7), фиолетовый (10), тёплый (15) |
| Фон поля | default (1), тёплый (5), прохладный (10), мятный (15), бумага (20) |
| Подсветка | flash (1), fade (1), solid (10) |
| Шрифт | Системный (1), JetBrains Mono (3), Roboto Mono (7), Source Code Pro (10), PT Mono (15) |
| Титул | Новичок (1), Ученик (5), Настойчивый (10), Мастер (15), Магистр (20) |
| Свой фон | 12 |

Шрифты подгружаются с Google Fonts; при недоступности CDN — fallback на системный monospace. Свой фон — JPG или PNG до 3 МБ, хранится на сервере (`static/uploads/backgrounds/`) с именем `{user_id}_{random_hex}.{ext}`. Валидация в четыре этапа: размер, magic-bytes сигнатура, `PIL.Image.verify()`, совпадение сигнатуры и того, что распознал Pillow.

## Компьютерное зрение

Базовая проверка: обе руки в кадре — иначе ввод блокируется. Дополнительные подсказки (жёлтый статус, не блокируют):

- **Руки перепутаны** — по `handedness` из MediaPipe + координата X запястья
- **Слишком близко / далеко** — расстояние между запястьями вне диапазона 15-75% ширины
- **Разверните ладони** — сильный вертикальный наклон (dy > dx)

Сообщения дебаунсятся на 500 мс. Пороги (`POSE_WRIST_MIN_DX`, `POSE_WRIST_MAX_DX`, `POSE_TILT_THRESHOLD`) вынесены константами в `index.html` — тюнить под конкретную камеру.

Если камера недоступна (отказ пользователя, занята другим приложением, нет устройства), приложение переходит в fallback-режим: печать разрешена, сверху жёлтый баннер «Камера недоступна» с раскрывающейся инструкцией. Настройка `cvEnabled` не меняется — при разрешении камеры и перезагрузке страницы CV снова попробует стартовать.

## Запуск

Нужен Python 3.10+ и ffmpeg-free окружение (без CV-зависимостей — только Flask, SQLAlchemy, Login и Pillow).

```bash
python -m venv .venv
.venv\Scripts\activate           # Windows
source .venv/bin/activate        # Linux/Mac
pip install -e .
python app.py
```

Открывай `http://127.0.0.1:5000`, регистрируйся.

Переменные окружения:

- `SECRET_KEY` — ключ подписи сессий. В dev работает дефолт; для прода сгенерируй: `python -c "import secrets; print(secrets.token_hex(32))"`.
- `DATABASE_URL` — строка подключения SQLAlchemy. По умолчанию `sqlite:///typist.db` в папке `instance/`.

## Тесты

```bash
pip install pytest pytest-check
pytest tests/ -v
```

107 тестов, покрытие `app.py` — 92%. CV-логика тестируется отдельным Node-скриптом:

```bash
node tests/test_pose_analysis.js
```

12 кейсов, проверяют `analyzePose` без live-камеры.

## Миграция БД

При запуске вызывается `db.create_all()` — создаёт отсутствующие таблицы, но **не добавляет колонки в существующие**. Если меняешь схему модели, удаляй `instance/typist.db` и перезапускай. Для прода — alembic.

## Известные ограничения

- Streak считается в UTC. Пользователь в Москве, закончивший сессию в 02:00 ночи, увидит «новый день» раньше, чем пользователь в Токио. Для учебного проекта — нормально.
- Нет rate-limiting на `/login`. Bruteforce технически возможен.
- Нет валидации никнеймов при регистрации (длина, символы). XSS закрыт автоматическим экранированием Jinja, но пустые или слишком длинные ники пройдут.
- Google Fonts — внешняя зависимость. При блокировке/медленной сети шрифт падает на системный monospace.
- MediaPipe может по-разному определять «левую/правую» руку в зависимости от камеры. Пороги консервативные, но могут потребовать калибровки.
